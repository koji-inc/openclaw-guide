    第1章：OpenClawとは何か¶

    "EXFOLIATE! EXFOLIATE!" — 宇宙ロブスター、たぶん

    1.1 OpenClawとは¶

    一言でいうと： OpenClawは、WhatsApp・Telegram・Discord・iMessageなど複数のチャットアプリを、AIエージェントにつなぐセルフホスト型ゲートウェイです。

    自分のマシン（Mac、Linux、Windows WSL2、VPS）でGatewayプロセスを1つ立ち上げるだけで、普段使っているメッセージアプリから「いつでもどこでもAIアシスタントに話しかけられる」環境が手に入ります。

    何ができるのか — 具体的なユースケース¶

    1. パーソナルAIアシスタント 専用のWhatsApp番号やTelegramボットを用意し、スマホからメッセージを送るだけでAIが応答する。外出先でもポケットの中にAIアシスタントがいる状態。

    2. マルチチャンネル統合 WhatsApp、Telegram、Discord、iMessage、Slack、Microsoft Teams、Matrix、Mattermost（プラグイン）など20以上のチャネルを1つのGatewayで同時に扱える。チャネルごとにバラバラのAIサービスを使い分ける必要がない。

    3. コーディングエージェント連携 内蔵のエージェントランタイム（Pi agent）がツール実行、ファイル操作、ブラウザ自動操作、Web検索などを行う。Discordで「このリポジトリのREADME書いて」と頼めば、実際にファイルを生成してくれる。

    4. モバイルノードによるリッチな連携 iOS/Androidアプリをノードとしてペアリングすると、カメラ撮影、画面録画、位置情報取得、Live Canvas（エージェントが動的に描画するHTML/CSS/JSキャンバス）、音声対話が可能になる。

    1.2 なぜOpenClawが必要なのか¶

    既存のAIツールとの違い¶

    ChatGPT、Claude.ai、Geminiなど既存のAIサービスは、基本的にそのサービスのWebサイトやアプリを開いて使うもの。便利だが、以下の制約がある：

    制約 既存サービス OpenClaw
    入口 専用アプリ/Webのみ 普段使いのチャットアプリから
    データ管理 サービス事業者のクラウド 自分のマシン
    ツール実行 サービス側の制限内 自分のマシンでコマンド実行可能
    カスタマイズ 限定的 SOUL.md等で自由にペルソナ定義
    マルチチャンネル 不可 1つのGatewayで複数チャネル同時対応
    モデル選択 そのサービスのモデルのみ 35+プロバイダー（Anthropic, OpenAI, Google等）

    セルフホスト型の意味とメリット¶

    OpenClawの「セルフホスト」は、Gatewayプロセスを自分のハードウェアで動かすということ。具体的なメリット：

    • データが外に出ない — 会話ログ、セッションファイル（.jsonl）はすべてローカルに保存される
    • 好きなモデルを使える — Anthropic、OpenAI、Google、あるいはOllama等のローカルLLMまで、OpenAI互換/Anthropic互換のエンドポイントなら何でも接続可能
    • 自分のルールで運用できる — allowlistで送信者を制限、ペアリング認証でDMを保護、ツールポリシーで実行範囲を管理
    • MITライセンス — オープンソースでコミュニティドリブン

    マルチチャンネルの価値¶

    「WhatsAppで聞いたことの続きをDiscordで」——OpenClawならそれが自然にできる。Gatewayがセッションの一元管理を行うため、チャネルをまたいでも文脈が維持される。

    エンジニア視点では、チームのSlackとプライベートのTelegramの両方から同じAIアシスタントにアクセスできるのが地味に強力。

    1.3 OpenClawのポジション¶

    類似ツールとの比較¶

    OpenClawは「メッセージングチャネル ↔ AIエージェント」のゲートウェイというユニークなポジションにいる。

    観点 OpenClaw 一般的なAIチャットボットフレームワーク
    焦点 パーソナルAIアシスタント（個人〜少人数向け） 顧客対応ボット（多数のユーザー向け）
    チャネル 20+のメッセージングアプリを同時サポート 通常1〜2チャネル
    エージェント 内蔵ランタイム（ツール実行、メモリ、マルチエージェント） 外部のAI APIを呼ぶだけが多い
    デプロイ セルフホスト（ローカルPCやVPS） クラウドサービスが多い

    OpenClawが得意なこと¶

    • パーソナルユース — 「自分専用のAIアシスタント」を作るのに最適化されている
    • チャネル統合 — WhatsApp、Telegram、Discord…使い慣れたアプリがそのままインターフェースになる
    • エージェントの自律性 — ファイル操作、コマンド実行、スケジュール実行（cron/heartbeat）まで対応
    • 拡張性 — Skills（スキルシステム）、Plugins（プラグインチャネル）、Lobster（ワークフローパイプライン）で機能を拡張できる

    OpenClawが苦手なこと（現時点）¶

    • 大規模マルチユーザー向けのボットプラットフォームではない — 基本的に個人〜少人数のパーソナルアシスタント用途
    • モデル自体は含まれない — AIプロバイダーのAPIキー（または自前のモデルサーバー）が別途必要

    1.4 OpenClawの全体像（ざっくり版）¶

    主要コンポーネント¶

    OpenClawのアーキテクチャは4つの主要コンポーネントで構成される：

    コンポーネント 役割
    Gateway 中心となるデーモンプロセス。すべてのチャネル接続、セッション管理、ルーティングを一元管理する。WebSocket APIを公開し、クライアントやノードと通信する。デフォルトで127.0.0.1:18789でリッスンする
    Agent 内蔵のエージェントランタイム（Pi agentコア）。モデル呼び出し、ツール実行（ファイル読み書き、コマンド実行、ブラウザ操作等）、ストリーミング応答を処理する
    Channel メッセージの入出力口。WhatsApp（Baileys経由）、Telegram（grammY経由）、Discord、iMessage、Slack等がビルトイン。Mattermost等はプラグインで追加
    Node macOS/iOS/Android/ヘッドレスのデバイス。WebSocketでGatewayに接続し、カメラ、Canvas、画面録画、位置情報などのデバイス固有コマンドを提供する

    加えて、Web Control UI（ブラウザダッシュボード）とCLIがGatewayへのWebSocket接続を通じて操作インターフェースを提供する。

    メッセージを送ったら何が起こるか — 5ステップで理解する¶

    例：TelegramからOpenClawに「今日の天気を教えて」と送った場合。

    [あなたのTelegram] → [Gateway] → [Agent] → [LLM API] → [応答をTelegramに返す]
    1. メッセージ受信 — TelegramチャネルがGatewayにメッセージを渡す
    2. ルーティング — Gatewayが送信者とチャネルに基づいてセッションを特定する（DMはmainセッション、グループは個別セッション）
    3. エージェント実行 — Agentがワークスペースのコンテキスト（AGENTS.md、SOUL.md等）とセッション履歴を組み立て、LLM（例：Claude）にリクエストを送る
    4. ツール実行（必要に応じて） — LLMがツール呼び出しを返した場合、Agentがローカルで実行する（天気スキルの実行、Web検索、ファイル操作など）。結果をLLMに戻し、最終応答を生成する
    5. 応答配信 — Gatewayがエージェントの応答をTelegramチャネル経由であなたに返す。長い応答はストリーミング＆チャンキングで分割送信される

    接続プロトコル¶

    GatewayとクライアントはWebSocket（JSONテキストフレーム）で通信する。接続時にconnectフレームを送り、認証トークン（設定時）とデバイスIDによるペアリングを行う。新しいデバイスは承認が必要で、ローカル接続（localhost）は自動承認が可能。

    1.5 まとめ¶

    この章で押さえておくべきポイント：

    • OpenClawは「チャットアプリとAIエージェントをつなぐセルフホスト型ゲートウェイ」
    • セルフホストだからデータは自分の手元、モデルも自由に選べる
    • 20+のチャネルを1つのGatewayで同時サポート
    • Gateway（中枢）→ Channel（入出力）→ Agent（頭脳）→ Node（手足）という構成
    • MITライセンスのオープンソースプロジェクト

    補足：OpenClawの名前の由来

    OpenClawの前身は「Warelay」（WhatsAppゲートウェイ）。その後「Clawd」→「Moltbot」と名前が変わり、2026年1月30日に現在の「OpenClaw」に落ち着いた。OPEN（オープンソース、誰にでも開かれている）+ CLAW（ロブスターのルーツ）。ロブスターが脱皮（molt）して成長するように、プロジェクト自体も名前を脱ぎ捨てながら進化してきた。
